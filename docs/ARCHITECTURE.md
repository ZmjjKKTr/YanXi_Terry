# 架构说明

> 用途：记录设计决策与分阶段计划，解释"为什么这么做"。

## 5 项设计决策

| # | 决策 | 结论 |
|---|------|------|
| 1 | 用户识别 | 架构预留多用户，`--user` 切换，per-user 数据隔离 |
| 2 | 知识库 | 共享基础库 500 条，profile 里 known_callers 白名单补充 |
| 3 | 转接通知 | 重要来电 4 轮事务转达后终端打印摘要，notify_user() 预留推送接口 |
| 4 | 学习策略 | 诈骗自动入库 + RAG 分类需人工质检确认 + 从 review.jsonl 增量学习 |
| 5 | 质检粒度 | 只检"系统犹豫过"的通话（高 score/纠偏/反向校验/兜底） |

## 分阶段计划

| 阶段 | 内容 | 状态 |
|------|------|------|
| 1 | 多用户基础设施（per-user 目录、白名单、诈骗入库） | ✅ |
| 2 | 重要来电多轮对话 + 转接通知 + 紧急分级 | ✅ |
| 3 | 通话质检 + 统计面板 + 增量学习闭环 | ✅ |
| 4 | 智能调度（上课代接/周末规则/决策转接）+ 便捷设置向导 | ✅ |
| 远期 | Web 界面（Streamlit） | 🔲 |

## Pipeline 处理流程

```
来电输入
  ├─ 重要来电多轮进行中？→ 是 → 继续多轮对话
  ├─ Gate 0a: 自动拦截规则
  ├─ Gate 0b: 来电白名单（known_callers）
  ├─ Gate 0c: 自动放行规则
  ├─ Gate 0d: 自动通知规则（不改变路由）
  ├─ Gate 1: 敏感词兜底拦截（28 个关键词）
  ├─ Gate 2: RAG 检索 + 阈值过滤（score > 1.2 → 兜底）
  ├─ Gate 2.5: 关键词纠偏（外卖/重要信号词强制修正）
  ├─ Gate 2.6: 反向校验（无信号词降级普通来电）
  └─ Gate 3: Agent 路由
       ├─ 外卖/快递 → 时间调度 + 决策转接检测
       ├─ 诈骗/推销 → 拦截 + 入库 scams.jsonl
       ├─ 重要来电 → 4 轮事务转达 + 紧急通知
       └─ 普通来电 → 通用兜底
```

## 技术选型

- **智谱 GLM-4-Flash**：中文理解好、成本低、API 稳定
- **Chroma + sentence-transformers**：轻量级本地向量库，无需外部服务
- **FastAPI**：异步支持、自动文档生成、Pydantic v2 类型校验
- **Vosk**：离线语音识别，不依赖网络
